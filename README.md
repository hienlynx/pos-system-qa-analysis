# POS Manual Testing Portfolio

This project demonstrates a structured manual testing approach for a Point of Sale (POS) system.

The objective is to analyze the business model of a POS system, identify potential business risks, and design a test strategy and test cases to ensure reliability of financial and inventory transactions.

---

## System Overview

A Point of Sale (POS) system is used by retail stores to process customer purchases, handle payments, generate receipts, and update inventory.

Core workflow:

Customer selects products  
↓  
Cashier adds items to cart  
↓  
Checkout  
↓  
Customer makes payment  
↓  
Receipt generated  
↓  
Inventory updated

---

## Testing Scope

The project focuses on testing critical POS transactions:

- Add Item to Cart
- Checkout
- Process Payment
- Refund Transaction
- Inventory Update

---

## Repository Structure

**01-business-analysis**  
Defines the business model including actors, entities, and core transactions.

**02-risk-analysis**  
Identifies business risks associated with POS transactions.

**03-test-strategy**  
Defines testing priorities and overall testing approach.

**04-test-cases**  
Contains detailed manual test cases for critical POS functionalities.

**05-test-data**  
Provides sample test data used in testing scenarios.

## Testing Focus Areas

1. Financial transaction integrity
